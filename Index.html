Academic Reference — Semester IV

This notebook contains compiled reference material for coursework across multiple subjects. Topics include data structures, algorithm analysis, operating systems, and computer networks. Notes are organized by unit and updated regularly throughout the semester.

Chapter 1 — Introduction to Computing

Computing systems are built on layers of abstraction — from physical transistors to high-level programming languages. Understanding these layers helps engineers write more efficient code and debug problems at the right level of the stack.

Chapter 2 — Data Structures

A data structure is a way of organizing and storing data so it can be accessed and modified efficiently. Common structures include arrays, linked lists, trees, graphs, stacks, and queues.

  • Arrays: O(1) access, O(n) search
  • Linked List: O(n) access, O(1) insert/delete at head
  • Binary Search Tree: O(log n) average case
  • Hash Table: O(1) average for search/insert
Chapter 3 — Algorithm Complexity

Big-O notation describes the upper bound of an algorithm's running time relative to input size. It helps compare algorithms independently of hardware. Constant factors are ignored — we care about growth rate as n approaches infinity.

Chapter 4 — Memory Management

Modern operating systems use virtual memory to give each process the illusion of its own address space. Pages are mapped to physical frames via a page table. When a page is not in memory, a page fault occurs and the OS must load it from disk.

Chapter 5 — Networks

The OSI model divides network communication into 7 layers. Each layer provides services to the layer above and uses services from the layer below. The TCP/IP model simplifies this to 4 layers: application, transport, network, and link.
